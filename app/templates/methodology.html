Open Practices Score — Methodology

How we measure open research practices at IGB — version 2, April 2026

1 Overview

The Open Practices Score estimates how consistently IGB researchers apply selected open research practices in their IGB-affiliated, OpenAlex-indexed publications. It combines five dimensions — open access, data openness, code openness, preprint sharing, and repository licensing — into a single composite indicator.

The score is calculated for current IGB staff with at least three indexed publications. Small-sample shrinkage is applied so that researchers with few publications are scored conservatively rather than penalised. The score is intended as a comparative benchmark, not a complete measure of all open science activity. Practices such as open peer review, open educational resources, and community engagement are not yet captured.

2 Eligibility

  • Current IGB staff listed in the IGB staff directory
  • At least 3 publications indexed in OpenAlex with an IGB affiliation
  • IGB-affiliated publications only — work published before joining IGB or with a different affiliation is not included
  • Authors with multiple OpenAlex IDs (e.g. name variants with/without umlauts) are merged automatically

3 Data Sources

  • OpenAlex Publication metadata, open access classification (gold/hybrid/green/bronze/diamond), publication type, citation counts. Updated weekly. openalex.org
  • FRED IGB's Freshwater Research and Environmental Database. 253 datasets with DOIs. fred.igb-berlin.de
  • DataCite DOI metadata for FRED datasets, including license/rights fields used to determine open licensing. api.datacite.org
  • Abstracts Text analysis of publication abstracts for data availability statements and code sharing signals

4 Scoring Dimensions

Each dimension captures a distinct open practice. A publication contributes at most once to each dimension. A FRED deposit is scored under Repository Licensing; data linked to publications is scored under Data Openness.

DimensionWeightWhat it measuresEvidence tierShrinkage n
OA Accessibility 30% How much of a researcher's output is openly readable Tier 1–2 total_pubs
Data Openness 25% Whether data behind publications is findable and reusable Tier 1 Tier 3 total_pubs
Code Openness 20% Whether analysis code is publicly available Tier 1 Tier 3 total_pubs

5 Evidence Tiers

The scoring is designed to reward verified, reusable openness more than stated intentions. Verified indicators receive 2–5× more credit than text-based signals.

TierDescriptionExamplesCredit
Tier 1: Verified Directly confirmable via metadata or DOI Open license in DataCite, OpenAlex OA status, dataset DOI, preprint record Full (1.0)
Tier 2: Strong proxy Reliable metadata but not independently verifiable OpenAlex OA category (hybrid vs green), dataset type classification High (0.8–0.9)
Tier 3: Inferred Text-based signal, may be false positive or negative Abstract mentions of GitHub, "data available at", "available upon request" Reduced (0.1–0.5)

6 Formula & Calculation

Empirical Bayes Shrinkage

Instead of a volume multiplier, each dimension's observed rate is adjusted using empirical Bayes shrinkage:

Adjusted_rate = (n / (n + k)) × observed_rate + (k / (n + k)) × IGB_mean_rate Where: n = publications relevant to that dimension k = 10 (smoothing constant) IGB_mean_rate = institutional average for that dimension

This means:

  • A researcher with 3/3 Gold OA doesn't get 100% — their rate is pulled toward the IGB average
  • A researcher with 80/100 Gold OA barely shrinks — the data speaks for itself
  • No researcher is penalised by a blanket multiplier; low-n scores are treated as uncertain, not bad

Final Score

Score = 0.30 × adj_OA + 0.25 × adj_Data + 0.20 × adj_Code + 0.10 × adj_Preprint + 0.15 × adj_Repo

Per-Publication Scoring

OA Accessibility (per publication):

Gold / Diamond = 1.0 (verified open journal) Hybrid = 0.9 (OA in subscription journal) Green = 0.7 (self-archived version available) Bronze = 0.2 (free to read, no open license — can be restricted anytime) Closed = 0.0 OA Rate = sum(per_pub_scores) / total_pubs × 100

Data Openness (per publication, maximum of applicable tiers):

Verified open dataset DOI linked in abstract = 1.0 (Tier 1) OpenAlex "dataset" publication type = 1.0 (Tier 1) Data availability statement in abstract = 0.3 (Tier 3) "Available upon request" only = 0.1 (Tier 3) None = 0.0 Data Rate = sum(per_pub_data_scores) / total_pubs × 100

FRED deposits are not counted here — they have their own dimension (Repository Licensing).

Code Openness (per publication, maximum of applicable tiers):

Verified software DOI (Zenodo, CRAN, PyPI) = 1.0 (Tier 1) Repository link in abstract (GitHub, GitLab...) = 0.5 (Tier 3) Generic code mention in abstract = 0.2 (Tier 3) None = 0.0 Code Rate = sum(per_pub_code_scores) / total_pubs × 100

Detected patterns: github.com, gitlab.com, bitbucket.org, codeberg.org, cran.r-project.org, pypi.org, "R package", "Python package", "source code", "code available".

Preprint Rate:

Preprint Rate = preprints / max(eligible_articles, 1) × 100, capped at 100 Eligible articles: research articles, reviews, letters Excluded: editorials, corrections, conference abstracts, book chapters

Repository Licensing:

Repo Rate = openly_licensed_FRED_deposits / max(total_FRED_deposits, 1) × 100 "Openly licensed" = CC-BY, CC0, CC-BY-SA, or similar in DataCite metadata No FRED deposits: shrinkage assigns the IGB mean for this dimension

9 Limitations

  • Abstract scanning is imperfect. Tier 3 indicators (code mentions, data statements) are detected via keyword matching in abstracts. This misses sharing mentioned only in the full text, and may produce false positives. Tier 3 signals receive reduced credit (0.1–0.5) to account for this.
  • Not all open practices are captured. Open peer review, open educational resources, open science mentoring, community engagement, preregistration, and FAIR compliance are not yet measured.
  • Only IGB-affiliated publications. Work published before joining IGB or with a different affiliation is not included.
  • Author disambiguation is imperfect. OpenAlex sometimes splits a single researcher into multiple IDs. We merge by name (with umlaut normalization), but some may still be missed.
  • FRED downloadability is checked via license metadata in DataCite. Some datasets may be practically accessible even without an explicit open license.
  • Citation differential is not included in the score. The OA citation advantage is heavily confounded (self-selection, field differences, sample size). It is reported as a contextual statistic but does not affect the composite score.

10 Design Against Gaming

Gaming riskCountermeasure
Adding "github" to abstracts without accessible code Verified software DOIs (Tier 1) get full credit; abstract mentions (Tier 3) get only 0.2–0.5
Depositing restricted FRED records for score points Repository dimension scores open-licensed deposits only; restricted deposits get zero
Splitting one dataset into multiple deposits Repository score is a rate (open/total), not a count — splitting has no effect
Chasing low-value preprints Preprints carry only 10% weight; shrinkage prevents small-n inflation
"Available upon request" as empty gesture Scored at 0.1, barely above zero

11 Current Benchmark Data (April 2026)

Only ~10% of FRED datasets are openly downloadable. Of 253 FRED datasets with DOIs, only 25 have open licenses (CC-BY, CC0, or similar). The remaining 228 are deposited but restricted. This is the single largest opportunity to improve IGB's open data practices.
IGB-wide average OA rate: ~67% (weighted by per-publication OA quality score). Gold OA accounts for 1,496 publications, hybrid 887, green 979, bronze 597, diamond 107.
Code sharing is rare. Only ~0.5% of publications have detectable code sharing signals in abstracts. This is likely an undercount (many share code in supplementary materials), but even so represents a major growth area.
Preprints are growing. 552 preprints from IGB researchers indexed in OpenAlex. The IGB-wide preprint rate is ~20% of eligible articles.

12 Future Improvements (v3)

  • Field normalisation: Show percentile rank within broad field categories (ecology, hydrology, social science)
  • Career-stage adjustment: Normalise by years since first publication, or show cohort percentiles
  • Better code/data detection: Cross-reference with Zenodo and GitHub APIs directly
  • FAIR compliance: Check metadata quality, persistent identifiers, and machine-readability
  • Preregistration: Add as a dimension when common enough at IGB
  • Sensitivity analysis: Vary weights ±5% and shrinkage k from 5–20 to test ranking stability
  • Manual audit: Sample 30–50 profiles to verify code/data detection accuracy
